NARASI IMPLEMENTASI ANTARMUKA (UI) SKRIPSI NETLABS

Platform Media Pembelajaran & Pendamping Praktikum Jaringan Digital Berbasis Hybrid RAG AI

a. Siswa (Aplikasi Mobile Android)

1) Halaman Login Siswa

Halaman antarmuka ini dirancang minimalis sebagai pintu utama verifikasi hak akses siswa. Komponen input difokuskan pada pengisian Nomor Induk Siswa (NIS) sebagai username serta kata sandi (password). Halaman ini dilengkapi dengan fungsi pesan kesalahan (error handling indicator) yang informatif dan responsif apabila proses autentikasi token Laravel Sanctum mengalami kegagalan (seperti kredensial tidak valid atau gangguan koneksi jaringan). Tampilan Halaman Login Siswa dapat dilihat pada Gambar 2.33 berikut.

(Tempatkan Gambar 2.33 di sini)
Gambar 2.33 Halaman Login Siswa


2) Halaman Beranda Siswa

Halaman Beranda berfungsi sebagai pusat informasi personal siswa yang menyajikan dashboard pembelajaran secara komprehensif. Pada antarmuka ini ditampilkan sapaan nama siswa secara dinamis, kartu rangkuman progres penyelesaian materi praktikum kejuruan, grafik capaian rata-rata nilai kuis, serta komponen infografis harian bertopik infrastruktur jaringan digital. Sistem secara otomatis melakukan validasi dan sinkronisasi data pembelajaran terenkripsi dari server sebelum diproses dan disimpan dalam sistem. Tampilan Halaman Beranda Siswa dapat dilihat pada Gambar 2.34 berikut.

(Tempatkan Gambar 2.34 di sini)
Gambar 2.34 Halaman Beranda Siswa


3) Halaman Detail Topik Materi

Halaman ini merupakan lembar bacaan digital interaktif yang menyajikan konten materi langkah kerja laboratorium jaringan (seperti teknik crimping kabel UTP/STP atau simulasi topologi jaringan pada Cisco Packet Tracer). Materi disajikan dalam struktur poin-poin ringkas (bulleted lists) dan kartu visual guna mengoptimalkan keterbacaan serta menghindari kepadatan visual (cognitive overload). Selain itu, halaman ini dilengkapi dengan tombol pintas (Floating Action Button / FAB) melayang untuk akses langsung menuju AI Tutor. Tampilan Halaman Detail Topik Materi dapat dilihat pada Gambar 2.35 berikut.

(Tempatkan Gambar 2.35 di sini)
Gambar 2.35 Halaman Detail Topik Materi


4) Halaman Chatbot (AI Tutor)

Antarmuka interaktif tanya-jawab ini dirancang sebagai pendamping siswa saat melaksanakan praktikum mandiri dengan memanfaatkan format gelembung percakapan (bubble conversation). Antarmuka ini dilengkapi dengan tombol pemutar audio Text-to-Speech (TTS) untuk mendengarkan jawaban AI, fitur unggah berkas/foto untuk analisis visual kendala konfigurasi perangkat, serta ikon indikator input perekaman suara (Voice Recognition). Tampilan Halaman Chatbot dapat dilihat pada Gambar 2.36 berikut.

(Tempatkan Gambar 2.36 di sini)
Gambar 2.36 Halaman Chatbot (AI Tutor)

b. Administrator & Guru (Aplikasi Web Back-Office)

1) Halaman Login Admin

Halaman autentikasi web back-office dirancang khusus bagi Administrator dan Guru untuk mengakses panel kontrol sistem. Antarmuka ini menerapkan mekanisme Role-Based Access Control (RBAC) yang memastikan bahwa hanya akun berhak akses tinggi yang dapat masuk. Halaman ini juga dilengkapi indikator proteksi sesi dan pesan validasi kredensial. Tampilan Halaman Login Admin dapat dilihat pada Gambar 2.38 berikut.

(Tempatkan Gambar 2.38 di sini)
